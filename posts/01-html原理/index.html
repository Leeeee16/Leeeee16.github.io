HTTP和HTTPS

HTTP全称是Hyper Text Transfer Protocol，中文为超文本传输协议，HTTP协议是用于从网络传输超文本数据到本地浏览器的传送协议，它能保证高效而准确地传送超文本文文档。HTTP由万维网协会（World Wide Web Consortium）和Internet工作小组IETF（Internet Engineering Task Force）共同合作制定的规范，目前广泛使用的是 HTTP 1.1 版本。

HTTPS的全称是Hyper Text Transfer Protocol over Secure Socket Layer，是以安全为目标的HTTP通道，简单讲是HTTP的安全版，即HTTP 下加入 SSL 层，简称为 HTTPS。

HTTPS所传输的内容都经过了SSL的加密，因此提高了传输的安全性。它主要由两种作用：

  1. 建立了信息安全通道，保证数据传输安全
  2. 确认网站安全性，使用HTTPS的网站通过浏览器地址栏上的小锁标志，来查看网站的认证信息等。

HTTP请求过程

在浏览器输入URL，回车后就可以在浏览器中看到页面。首先是浏览器向网站的服务器发送一个请求，服务器接收请求后进行处理和解析，然后返回对应的响应并传回给浏览器。浏览器会对响应进行解析，包括页面的源代码等内容，最终将页面呈现在浏览器上。

请求

请求是客户端向服务端发出的，分为：请求方法（Request Method）、请求的网址（Request URL）、请求头（Request Headers）、请求体（Request Body）。

请求方法
最常见的两种请求方法：GET 和 POST。 GET，就是拿到，得到。比如在浏览器中输入一个URL并回车，这就是一个GET请求，请求的参数会包含在URL中。
POST，有投递的意思。比如在登录网址时，输入用户名和密码并点击登录，这样我们就发起了一个POST请求。

两个方法的区别：

  • GET请求中的参数包含在URL中，可以直接看到，但在POST请求中，URL中并不会包含这些数据，数据是采用表单的形式传输的，会包含在请求体中。
  • GET请求提交的数据最多是1024字节，而POST没有限制。

请求头
包括Cookie、Referer、User-Agent等。

  • Cookie(s):这是网站为了辨别用户进行会话跟踪而存储在用户本地的数据。
  • Referer：用来标识这个请求是从哪个页面发来的，服务器可以拿到这一信息并做响应处理，如来源统计、防盗链处理等。
  • User-Agent：可以让服务器识别客户使用的操作系统、浏览器版本等等。在设计爬虫时，加上此信息，可以伪装浏览器。否则，我们的爬虫很可能被识别出来。

Session与Cookies

  • Session 中文意思是会话。当客户端第一次请求服务器时，服务器会返回一个响应头中带有Set-Cookie字段的响应给客户端，用来标记是哪一个用户，客户端浏览器会把 Cookies 保存起来。当浏览器下一次再请求该网站时，浏览器会把此 Cookies 放到请求头一起提交给服务器，Cookies 携带了 Session ID 信息，服务器检查该 Cookies 即可找到对应的 Session 是什么，然后再判断 Session 来以此来辨认用户状态。
    在成功登录某个网站时，服务器会告诉客户端设置哪些Cookies信息，在后续访问页面时客户端会把Cookies发送给服务器，服务器再找到对应的Session加以判断。如果Session中的某些设置登录状态的变量是有效的，那就证明用户处于登录状态，此时返回登录之后才可以查看的网页内容，浏览器再进行解析便可以看到了。
    反之，如果传给服务器的 Cookies 是无效的，或者 Session 已经过期了，我们将不能继续访问页面，此时可能会收到错误的响应或者跳转到登录页面重新登录。

  • 关闭浏览器后，Session会消失吗？
    不会，Session是在服务器端的，当我们关闭浏览器后，服务器并不知道浏览器已经关闭。因此如果我们关闭浏览器后，再次使用原来的Cookies仍然能够找到对应的Session ID，从而保持登录状态。

  • 服务器可以为Session设置一个失效时间，当距离客户端上一次使用Session的时间超过了失效时间，服务器就可以删除Session以节省空间。
